# ADR-003 — Plugins são adaptadores de stack

## Contexto

A plataforma precisa suportar múltiplas tecnologias mantendo comportamento idêntico entre elas.

Sem uma restrição clara, implementações específicas de linguagem tendem a incorporar regras de engenharia (ex: tratamento de cobertura ou lógica de falha), criando divergência entre stacks e tornando a plataforma inconsistente.

Isso faria com que o mesmo workflow produzisse resultados diferentes dependendo da tecnologia utilizada.

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Decisão

Plugins são tratados exclusivamente como adaptadores técnicos de execução.

Eles recebem uma intenção já decidida pela engine e executam comandos da stack correspondente.

Plugins não contêm regras de engenharia.

Responsabilidades permitidas:

* executar testes
* coletar cobertura
* publicar artefatos

Responsabilidades proibidas:

* decidir falha ou sucesso do pipeline
* interpretar branch ou versão
* validar commits
* aplicar políticas organizacionais
* alterar comportamento baseado em tecnologia

Toda decisão pertence à camada shared/engine.

## Consequências

Positivas:

* comportamento uniforme entre stacks
* fácil adição de novas linguagens
* menor risco de regressões cruzadas
* previsibilidade operacional

Negativas:

* menor liberdade na implementação do plugin
* necessidade de adaptação da engine para novos casos

Riscos aceitos:

* mais lógica centralizada em troca de consistência global
