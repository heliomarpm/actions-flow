# ADR-001 — Separação entre Política e Execução

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

A plataforma precisa suportar múltiplas stacks sem que regras de engenharia dependam da tecnologia utilizada.

Misturar validações de engenharia (ex: cobertura, promoção de branch, versionamento) com comandos específicos de linguagem torna o sistema frágil, difícil de evoluir e impossível de manter consistente entre projetos.

Também impediria adicionar novas stacks sem alterar comportamento existente.

## Decisão

A arquitetura separa explicitamente:

* **Política (engine/shared):** decide o que deve acontecer
* **Execução (plugins):** executa comandos da stack

Plugins não podem:

* validar commits
* interpretar branches
* bloquear pipeline
* decidir versão
* aplicar regras organizacionais

Plugins apenas respondem à plataforma executando:

* testes
* geração de cobertura
* publicação

## Consequências

Positivas:

* novas stacks podem ser adicionadas sem alterar regras
* comportamento consistente entre tecnologias
* versionamento previsível
* menor risco de regressões

Negativas:

* exige disciplina arquitetural
* algumas implementações ficam mais indiretas

Riscos aceitos:

* maior verbosidade interna em troca de estabilidade comportamental
