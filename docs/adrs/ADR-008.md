# ADR-008 — Convenção acima de configuração

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

Ferramentas altamente configuráveis tornam-se difíceis de entender, documentar e manter.

Cada novo parâmetro aumenta combinações possíveis e dificulta prever comportamento do sistema.

Para manter previsibilidade, a plataforma deve favorecer comportamento padrão estável ao invés de personalização extensa.

## Decisão

A plataforma prioriza convenções estáveis sobre novas opções configuráveis.

Novos inputs só são adicionados quando:

* não existe inferência segura possível
* o comportamento muda semanticamente
* múltiplos projetos reais exigem a variação

Configuração não deve substituir design.

## Consequências

Positivas:

* menor complexidade cognitiva
* documentação mais simples
* comportamento previsível
* menor custo de manutenção

Negativas:

* menor flexibilidade para casos específicos
* necessidade de adaptação do usuário ao padrão

Riscos aceitos:

* restrição de customização em troca de consistência sistêmica
