# ADR-005 — Promoção baseada em níveis de estabilidade de branch

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

Projetos frequentemente tratam branches apenas como convenção social, sem garantia operacional de estabilidade.

Isso causa:

* merges inseguros
* releases inconsistentes
* divergência entre ambientes
* dependência de revisão manual para validar maturidade do código

Para que a plataforma imponha previsibilidade, cada branch precisa representar um nível claro de estabilidade.

## Decisão

Branches representam níveis progressivos de confiança.

O código não é promovido por decisão manual, mas por satisfação de garantias técnicas.

Modelo padrão:

* develop → código em validação
* release → candidato a versão
* main → estado estável publicado

A promoção entre níveis só ocorre quando:

* quality gate aprovado
* versão válida calculada
* regras de governança satisfeitas

Não existe promoção baseada apenas em aprovação humana.

## Consequências

Positivas:

* redução de erro operacional
* previsibilidade de releases
* pipelines autoexplicativos
* alinhamento entre engenharia e operação

Negativas:

* menor flexibilidade para exceções
* necessidade de adaptação inicial do fluxo de trabalho

Riscos aceitos:

* rigidez operacional em troca de confiabilidade organizacional
