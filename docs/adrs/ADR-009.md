# ADR-009 — Execução determinística

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

Pipelines que variam comportamento entre execuções tornam-se difíceis de confiar e impossíveis de depurar.

Dependências implícitas, heurísticas instáveis ou ordem não garantida de execução geram resultados inconsistentes.

Uma plataforma de engenharia precisa produzir o mesmo resultado para a mesma entrada.

## Decisão

A execução do pipeline deve ser determinística.

Mesma branch + mesmo commit + mesmos inputs → mesmo resultado.

Não é permitido:

* comportamento baseado em arquivos não declarados
* decisões baseadas em ordem não controlada
* heurísticas não registradas
* dependências externas implícitas

## Consequências

Positivas:

* reprodutibilidade
* debugging confiável
* previsibilidade operacional
* segurança em automações

Negativas:

* necessidade de validações adicionais
* restrições em otimizações oportunistas

Riscos aceitos:

* menor flexibilidade em troca de confiabilidade sistêmica
