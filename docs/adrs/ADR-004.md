# ADR-004 — Versionamento determinístico baseado em histórico

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

Versionamento manual é uma das principais fontes de inconsistência em projetos de software.

Decidir versão com base em julgamento humano introduz:

* erros de incremento
* divergência entre changelog e código
* inconsistência entre projetos
* insegurança em automações

Para que a plataforma seja previsível, a versão precisa ser derivada exclusivamente do histórico do repositório.

## Decisão

O versionamento da plataforma é determinístico e baseado em histórico de commits.

A versão é calculada automaticamente a partir de:

* convenção de commits
* estado da branch
* histórico consolidado

Nenhuma decisão manual de versão é permitida dentro do fluxo padrão.

O mesmo commit sempre gera a mesma versão.

Alterações no cálculo de versão são consideradas mudanças de comportamento e seguem versionamento SemVer da própria plataforma.

## Consequências

Positivas:

* releases previsíveis
* eliminação de erro humano
* consistência entre múltiplos projetos
* changelog gerado de forma confiável

Negativas:

* dependência de disciplina na escrita de commits
* menor flexibilidade para ajustes manuais pontuais

Riscos aceitos:

* maior rigor operacional em troca de confiabilidade sistêmica
