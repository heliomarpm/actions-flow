# ADR-002 — Workflows são contratos públicos versionados

**Status:** Accepted
**Data:** 2026-02-14
**Decisores:** Maintainers do projeto

---

## Contexto

Os workflows desta plataforma são reutilizados por múltiplos repositórios e passam a fazer parte da infraestrutura desses projetos.

Alterações silenciosas em inputs, defaults ou comportamento podem quebrar pipelines existentes sem alteração no código consumidor, tornando o sistema imprevisível e inseguro para adoção contínua.

## Decisão

Cada workflow é tratado como uma API pública versionada.

Isso implica:

* Inputs são contratos estáveis
* Mudança de comportamento é breaking change
* Mudança de default é potencial breaking change
* Remoção de input exige depreciação prévia
* Novos comportamentos devem ser opt-in

Versionamento segue SemVer:

* **MAJOR**: quebra de contrato
* **MINOR**: nova capacidade compatível
* **PATCH**: correção interna sem alteração de comportamento

## Consequências

Positivas:

* usuários podem confiar em atualização automática
* adoção segura em múltiplos projetos
* previsibilidade operacional
* facilita manutenção a longo prazo

Negativas:

* evolução mais cuidadosa
* necessidade de manter compatibilidade temporária

Riscos aceitos:

* crescimento controlado de complexidade em troca de estabilidade externa
